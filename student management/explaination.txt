 
3 Layers connecting Method:
   Frontend (HTML, CSS, JavaScript): Displays the user interface and collects user input. It sends requests to the backend using the Fetch API.
Backend (Python + Flask): Receives the requests from the frontend, validates the input, performs the required business logic, and communicates with the MySQL database through REST APIs.
Database (MySQL): Stores all student records permanently. The backend executes SQL queries to insert, retrieve, update, delete, and search student data.


Hardest part and how i resolved it:
   The most challenging part was connecting the Flask backend with the MySQL database and ensuring that all CRUD operations worked correctly. Initially, I encountered issues with installing Flask and configuring the database connection.
   I resolved these by installing the required Python packages, verifying the MySQL server was running, creating the database and table correctly, and updating the database configuration with the proper MySQL credentials. After testing each API endpoint individually, the frontend was able to communicate successfully with the backend and database.
